# Overview

Deployment setup for h21 on an Ubuntu VPS using `uv` as the Python project
manager.

## Changes made

- Added `.python-version` (3.13) so `uv` auto-installs the right Python.
- Added `uv.lock` (generated by `uv sync`) for reproducible dependency
  resolution.
- Added `h21-reset` script entry point (`src/h21/reset.py`) that deletes
  database and log files from the data directory — a stop-gap for schema
  upgrades until proper migrations exist.
- Updated `cli()` in `main.py` with `--public` flag (binds to `0.0.0.0:80`)
  and optional `--host`/`--port` overrides. Bare `uv run h21` defaults to
  `127.0.0.1:8000`.
- Rewrote `ubuntu-setup.md` to use `uv` throughout (install, sync, run,
  update, reset).

## Usage

```bash
uv sync                              # install deps + create venv
uv run h21                           # start server (localhost:8000)
uv run h21 --public                  # start server on 0.0.0.0:80
uv run h21 --public --port 8080      # override just the port
uv run h21-reset                     # delete db + logs
```
